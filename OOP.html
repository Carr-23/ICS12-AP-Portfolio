What is OOP?

Definition: OOP Stands for Object Oriented Programming.

Procedure Vs. OOP: Proc programming uses only ONE program to run the code, unlike OOP that uses a combination of multiple classes all combined.

OOP Breakdown:
  • A Class is like the template used to make something.
  • Object is the thing made out of the class aka template
  • Attributes are the characteristiks of the object, aka the variables
  • Behaviours are the actions that can be done by the object, aka the methods

Global Variables in a class must be kept private that way other classes don't accidentally change its values. To acces these variables we have to manually get and change their values using 2 types of methods.

  • Accesor Methods, gets a value from a class that can normally not be accesed
  • Mutator Methods, sets a value from a class that can normally not be changed

Firstly, to create an object a constructor is needed. A constructor is like a method. It has the same name as the class. However it is not a mutator method. Inside the constructor is where the default values for the object are set. More than one constructor can be made but they must have a different parameters list that defines that object. This is called an overloaded constructor since the both have the same name and the parameters values are the only thing telling them apart. If a constructor is not made, Java will create a default constructor.

Let's say you forget to include a vairable in your class or maybe a method, well worry not this can be solved all thanks to inheritens. What you can end up doing is extending the class, this will create a subclass (The class that was extended will be called the superclass). The subclass won't have to define any of the variables or methods again since they are inherited from the superclass. Although you don't need create the methods again, the subclass can override one of the methods in the superclass. Once we have a couple of classes being extended from one another, we can create a Class Hierarchy Tree to visualize the break down from the original object class.

Up Casting and Down Casting is a way to travel through the class hierarchy tree. Upcasting is going up to the superclasses. This can never fail because once an object is created, anything above it already is created with it. On the other hand, downcasting can fail. If an object is trying to go down the hierarchy it can fail because more than one subclass could have been created and Java wouldn't know which one to go to.
