What is Recursion?

 

Recursion is the process of calling oneself while already being in it (Calling a method while already being inside that method). The process allows solutions that are repetitive to be solved efficiently. Recursion will use less code and will be more simplistic than creating multiple functions. A recursive function must eventually come to an end, otherwise a stackoverflow will occur. A recursive method almost always have a parameter. This allows a new value to be passed each time it is called allowing for a different calculation or a continuation of a previous calculation.

Ex.

public String loop (int x){

        if (x == 4)

                return "Done";

        else

                return loop(x+1);

}

There are three kinds of recursion, Linear, Binary and Tail.

Linear: A Linear Recursion refers to a recursion that only calls itself once inside the method.

Binary: A Binary Recursion refers to a recursion that calls itself more than once inside the method.

Tail: A Tail Recursion refers to a recursion that has the call statement located at the very end of the method.

Recursion is similar to a for or while loop. It repeats itself until the end has been reached. A while loop is an infinite loop. It repeats itself until the loop statement isn´t met hence the name while. A for loop works differently. It doesn´t go on forever. Instead it goes on until the amount of loops requested has occured. Out of both loops, recursion is closest to a while loop. However, instead of checking to make sure that the statement is met, it waits until an end statement is found. This end statement is most likely the first return statement that actually gives out a value.
